What should the app feel like to touch and use?

Interactions barely move. Hover changes opacity or background, nothing more. The content is always the focus — UI never draws attention to itself.
Buttons: opacity fade on hover
List rows: background tint only
Page transitions: instant or 100ms fade
Modals: simple fade in
B — Spring Physics

RSI Mean Reversion
Every interaction has a physical snap. Buttons scale up on hover, compress on press. Lists shift slightly. The app feels alive and responsive — like it wants to be touched.
Buttons: scale 1.04 on hover, 0.96 on press
List rows: subtle translate + tint
Page transitions: slide + spring (200–300ms)
Cards: lift on hover (shadow + scale)
C — Purposeful Moments

UI is mostly still, but meaningful moments get intentional animation — a backtest completing, a result revealing, a number counting up. Stillness makes the motion feel earned.
Idle UI: nearly still (Option A)
Backtest completes: results count up
New result card: slides in from below
Key metrics: scale on focus/hover

My lean: Option C — stillness as the default makes the moments of animation feel intentional and delightful, not noisy. A backtest result appearing should feel like a reveal, not just a page load. Pairs perfectly with the Notion Pure palette.
