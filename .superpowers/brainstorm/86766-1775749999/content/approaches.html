Tech Stack Approaches

Three ways to build a cross-platform (web + mobile) backtesting app

A

Flutter — Single Codebase

One Dart codebase targets iOS, Android, and Web. FastAPI Python backend handles heavy backtesting.

Pros

  • True single codebase
  • Excellent mobile performance
  • Rich chart/visualization widgets
  • Fast hot-reload dev cycle

Cons

  • Dart learning curve
  • Monaco/Python editor support is limited
  • Smaller ecosystem than React

Stack: Flutter · Dart · FastAPI · PostgreSQL · Redis

B

Next.js + React Native (Monorepo) — Recommended

Shared TypeScript business logic. Next.js for web (Monaco code editor). Expo/React Native for mobile. FastAPI Python backend.

Pros

  • Monaco editor (VS Code quality) on web
  • Best-in-class web performance & SEO
  • Huge JS/TS ecosystem
  • Shared packages reduce duplication

Cons

  • Two frontend targets to maintain
  • More complex monorepo setup

Stack: Next.js · Expo/RN · TypeScript · FastAPI · PostgreSQL · Redis

C

React Native Web — Shared UI

React Native components that render to both DOM and native. One UI codebase. FastAPI Python backend.

Pros

  • Maximum code sharing (UI + logic)
  • Single component library

Cons

  • React Native Web is not production-mature
  • Charts/complex layouts degrade on web
  • Limited Monaco editor support
  • Styling is harder to get right

Stack: React Native Web · TypeScript · FastAPI · PostgreSQL · Redis

👆 I recommend Option B — it gives you the best Python code editor experience on web (critical for quant users) while still shipping a polished native mobile app.
